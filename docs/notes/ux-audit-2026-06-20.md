# Weblab UX Audit — 2026-06-20

**Method:** Code-read UX assessment of the core first-time-user journey, run as a
fan-out of 7 sonnet sub-agents (one per flow) using the `/ux-assesment`
heuristics (clarity, cognitive load, friction, discoverability, feedback,
consistency, speed-of-understanding). Synthesized with the bug/UX findings logged
across the `/loop` QA campaign (BACKLOG "QA loop" entries iters 1–5).

**Access constraint (why this is code-read, not click-tested):** this session had
no Chrome extension connected (live browser QA unavailable) and the weblab-agent
API returned `AUTH_FAILED`. Findings are inferred from components, not observed in
a running app — treat severities as well-reasoned estimates, confirm the top ones
live before large redesigns.

**Scope note:** usability only. Screen-reader / WCAG / ARIA accessibility was
explicitly out of scope.

---

## Executive summary

| Severity | Count | Theme |
|----------|-------|-------|
| Blocker | 4 | First-time users can't get oriented or lose typed input; returning users hit a blank boot; CMS workflow is inverted |
| Frustrating | ~22 | Hidden affordances (icon-only rails, mode toggles), unlabeled controls, missing feedback, copy that doesn't match context |
| Minor | ~14 | Native `title` tooltips, jargon labels, missing time anchors, two-step flows that could be one |

**The single biggest theme: discoverability.** The product is deep (responsive
breakpoints, modes, CMS, publish targets) but most power lives behind unlabeled
icons, hover-only tooltips, and selection-gated toolbars. A first-time user can
finish a whole session unaware that breakpoints, Pan/Comment modes, or the CMS
even exist.

**Already fixed during this campaign** (so not re-listed as open): provisioning-
failure error routing, clone-dialog error UX, blank-create loader phase, bg
gradient→image clear, plain-text file highlighting, number-input blur-commit
guard, page-delete confirmation, commit-Continue gating, CMS sync/test Set
concurrency, **CMS partial-sync warning toast (this iter)**, **CMS field-reorder
double-click guard (this iter)**, **publish Live/Update label baseline (this
iter)**.

## Frustrating (high-value, mostly quick wins)

**Landing / onboarding**
- Mode toggle reads "Build" in low-contrast muted text with no chevron — newcomers don't notice a mode exists or that "Plan" routes elsewhere. Add a `ChevronDown`, bump contrast to `text-foreground-secondary`. `chat-mode-toggle.tsx:85`. *quick-win*
- Auth modal title "Sign in to edit" is wrong in the **create** context (last gate before signup). Change copy to "Sign in to start building". `messages/en.json` + `auth-modal.tsx:27`. *quick-win*
- Suggestion chips overwrite typed content behind a native `window.confirm()` — jarring vs the app's polish. Replace with a toast-with-undo or AlertDialog. `hero/create.tsx:373`. *quick-win*
- Post-auth resume lands on `/projects/new` (flash of unrelated UI) before auto-submit. Return to `/` and resume there. `create-draft.ts` + `hero-v2.tsx`. *structural*
- Blank-project loader shows no time expectation; users assume it's broken after ~20s. Add "This usually takes 15–30 seconds." `project-creation-loader.tsx:62-67`. *quick-win*
- AI-creation stall watchdog fires at 120s vs 25s for blank — lower to ~45s so anxious users get reassurance sooner. `hero/create.tsx:241`. *quick-win*

**Editor core**
- **Left rail is 8 unlabeled icons** (Insert/Components/Layers/Search/Brand/Pages/Images/Branches); labels only on hover; the hover-peek vs click-to-pin model is hidden. Add persistent labels or a label strip + a visible pin affordance. `left-panel/design-panel/index.tsx:277-334`. *structural*
- **Pan & Comment modes are buried in the bottom bar**, split from the top-bar Design/Code/CMS toggle, and the bottom bar vanishes in Code mode. Consolidate or label as a tool group + first-run hint. `bottom-bar/index.tsx` + `top-bar/mode-toggle.tsx`. *structural*
- Right panel **defaults to Chat**, so a design-tool user lands on a chat surface; the Style empty-state ("Select an element…") is never seen. Default to Style (showing the affordance) or add a hint in the Chat empty state. `right-panel/index.tsx:157`. *quick-win*
- **Style/Interactions tabs are disabled in Code mode with no path back** — clicking them does nothing; user must hunt for the top-bar toggle. Make the disabled triggers switch back to Design on click. `right-panel/index.tsx:437-513`. *quick-win*
- Left-panel collapse **also collapses the right panel** (tooltip says "Collapse panel", singular); the right panel's own close button is commented out. Restore it or fix the tooltip to "Collapse both panels". `left-panel/design-panel/index.tsx:288` + `right-panel/index.tsx:322`. *quick-win*
- Onboarding tour fires before the canvas iframe has content and doesn't explain element selection or the left rail. Gate the canvas step on `readyState.sandbox`; add a left-rail step. `onboarding-tour.tsx`. *structural*

**Responsive**
- **Responsive editing is invisible until a frame is selected** — the whole feature lives in the per-frame TopBar dropdown + "+", both hidden when nothing is selected. Surface a persistent "Breakpoints" entry. *structural*
- **Active breakpoint is unlabeled while editing** — a user editing padding on the Phone frame has no reminder they're writing Phone-scoped styles. Add an always-visible "Phone · 375px" chip in the editor bar (value already in `editorEngine.breakpoints`). *structural*
- "+" add-breakpoint hidden on non-rightmost frames; add-menu omits the canonical Desktop/Tablet/Phone presets. Show "+" on any selected frame; prepend `DEFAULT_BREAKPOINT_PRESETS`. `canvas/frame/top-bar/index.tsx:366,394`. *quick-win*
- Drift dot + override alt-click reset communicated only via native `title` (invisible on touch). Use `HoverOnlyTooltip` + a small reset chip. `canvas/frame/top-bar/index.tsx:337`, `editor-bar/inputs/override-affordance.tsx:57`. *quick-win*

**Publish / commit / share**
- **Publish button returns `null`** when caps are loading or absent — looks like a render bug; free/read-only users never learn publish exists. Render a disabled button + tooltip/upsell. `top-bar/publish/index.tsx:26`. *quick-win*
- Commit button has **no pending-changes indicator** — no nudge to commit before deploy. Badge it from git status. `git-actions.tsx`. *quick-win*
- Commit-vs-publish relationship is unexplained; the "+" share/invite button is unlabeled and ambiguous; access settings are disconnected from the invite popover. Add helper copy + a "Share" label + a "Manage access" link. *quick-win / structural*
- Deployment **failure is passive** — no toast, user must notice the button color and expand the dropdown. Fire `toast.error` on `DeploymentStatus.FAILED` + inline retry. `dropdown/preview-domain-section.tsx`. *quick-win*

**CMS**
- Fields tab is a **dead end** with no collection selected (bare text, no CTA). Add a "Go to Collections" button. `cms-workspace/fields-tab.tsx:156`. *quick-win*
- Sources empty state ("Add a collection to seed the default source") is confusing; Connect-source Save is disabled until Test passes with no explanation; map-collections dialog is jargon-heavy and auto-opens. Copy + a tooltip on the disabled Save + a help line. *quick-win*

**Reliability/feedback (cross-cutting)**
- `CanvasErrorBoundary fallback={null}` silently removes a crashed panel; the default copy wrongly blames the sandbox. Render a minimal inline error + Reload; fix the generic copy. `main.tsx`, `canvas-error-boundary.tsx:59`. *quick-win*
- Chat tab spins forever if the messages query errors. Add an error+retry state. `chat-tab/index.tsx:27`. *quick-win*

## Minor (polish)

Native `title` tooltips that should be real tooltips (New-item disabled, drift dot, override reset, items-table); jargon labels ("Freestyle" → "Weblab Hosting", "Routing" → "Detail page", Size "Behavior" group); the `/sign-up` link round-trips back to `/sign-in`; hardcoded English in the stall caption (`hero/create.tsx:416`, has a `// TODO(i18n)`); framework-select dialog interrupts "Start blank" for newcomers; bind-dialog bare "Loading…"; commit "Next steps" actions lack descriptions; hosting-integrations token hint isn't a link + no clear Test→Save success state; deploy progress bar has no log link.

## Biggest UX opportunities (synthesis)

1. **Make depth discoverable.** Label the left rail, surface responsive +
   modes, and give the onboarding tour one more pass — the product is more
   capable than it looks, and that gap is the #1 first-run risk.
2. **Never lose user input.** B2 (typed prompt discarded) is the sharpest
   instance; the campaign already fixed several editor-side input-loss bugs.
   Audit every "navigate away / sign-in bounce" for draft preservation.
3. **Feedback on the slow + failing paths.** Boot, provisioning, publish, and
   sync all have moments where the user can't tell "working" from "broken."
   Time anchors, non-null disabled states, and proactive failure toasts.
4. **Match copy to context.** "Sign in to edit" at create time, "Freestyle",
   "Routing", "Build" — small wording fixes that cut cognitive load at decision
   points.

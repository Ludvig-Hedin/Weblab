## When to add a guide

Add a guide when the same question would otherwise be answered ad-hoc in chat or PR comments more than once. A guide should:

- Have a single, specific outcome (e.g., "deploy the docs site to Railway").
- Start from a clear precondition (what state the reader is in).
- End with verification (how the reader knows it worked).

Architecture explanations (the *why*) go in [`../agent-context/`](../agent-context). Long-form planning goes in [`../product/`](../product) or [`../superpowers/`](../superpowers). Day-to-day debugging notes go in [`../notes/`](../notes).
